三、模块

1.含义:

在Python中一个**.py文件就是一个模块，里面定义了一些函数和
变量，需要的时候可以导入并使用这些模块*。
执行步骤：
1.在python模块加载路径中查找相应的模块文件
2.将模块文件编译成中间代码
3.执行模块文件中的代码

2.分类：

①内置模块：如：random、time、os、logging,直接导入即可使用。
②第三方模块（第三方库）：需要利用pip install下载包进行解压安装，但要注意安装环境和路径配置。种类丰富，功能多样。需要适当积累。

③自定义模块：即自己在项目中定义的模块，相当于自己写一个.py文件，用另一个代码来import导入调用它。但是注意不要和内置模块重名，会报错。

3.导入方式：

①import导入：如import numpy
②from xxx import xxx：从指导模块导入指定部分。
from 模块名 import 功能1,功能2…（功能可以是函数或者变量等）
③from xxx import *** ：意思是将模块中的所有功能全部导入。
④利用as给模块起别名：
import numpy as ny后，就可以用“ny.功能”来调用模块中的功能**；
或者from xxx import function A as f_a后，直接用f_a来调用function A。

  • 导入的模块可以直接调用里面的变量和函数。
    直接import导入的使用方式：模块名.内部函数（或变量）。
    from xxx import 导入的使用方式是;直接函数或者变量名。
  • from xxx import与直接import相比，使用功能时会少写一些文本。
    但是不建议过多使用from xxx import这种方式导入模块，因为如果from import进来的模块中函数或变量命名有冲突会造成错误。
    所以多用import来导入，使用功能时加上模块名。

4.内置全局变量____name____

__name__是 Python 的一个内置变量：

  • 当文件被 直接运行 时，__name__ 的值是 "__main__"。
  • 当文件被 作为模块导入 时，__name__ 的值是该模块的名字（即文件名去掉.py）。

作用：控制.py文件在不同的应用场景下执行不同的逻辑

  • 语法1：if name == “main“:

    使用情景：文件本身是执行主体，自己执行自己，而不是被导入调用的。
    通过使用__name__变量，**可以在模块底部添加测试代码，这些代码只会在模块被直接运行时执行，而不会在模块被导入时执行。**这种做法使得编写既可以作为脚本运行又可以作为模块导入的Python文件变得非常简单。

  • 语法2：if name == “模块名” :
    使用情景：这种用法很少见，一般只在调试或特殊逻辑中使用。

四、包

概念：包就是项目结构中的文件夹/目录，但是包的本质依然是模块，包也可以包含着另一个包（类似文件夹嵌套？）。

作用：
包就是将有联系的模块放到同一个文件夹下，并且在这个文件夹中创建一个名字为__init__.py文件，那么这个文件夹就称之为包，有效避免模块名称冲突问题，让结构更清晰。

新建包：右键项目名->new->PythonPackage

1.模块与包：

①导入包的方式：import + 包名（就是文件夹名）。
利用import导入包时，首先执行__init__中的代码。

②__all__变量：一个列表，可以控制要引入的东西(模块、函数、类等)，列表内的元素包含着要导入的所有模块。
如果在__init__.py中创建这个变量，放入所有带导入的模块，到时候就可以直接from package01 import *来直接导入__all__列表中的所有模块。
如果不设置__all__列表，则*无意义，导入不了任何模块，只能一个一个导入。
__all__变量相当于设置了一个准许导入模块的列表。

2.闭包

闭包是一种特殊的函数。
函数名种保存着函数所在位置的引用。

2.1 闭包的条件
闭包的本质是一个函数对象，它能够记住并访问其定义时所在作用域的变量，即使该作用域已经不存在了。要形成闭包，必须同时满足以下三个条件：

  1. 存在函数嵌套：内部函数必须定义在外部函数内部
  2. 内部函数引用外部函数的局部变量：这些变量称为自由变量（free variable）
  3. 外部函数返回内部函数对象：而不是直接调用内部函数

在这个例子中，当我们调用outer(10)时，outer函数执行完毕并返回inner函数对象。此时outer函数的作用域已经结束，但当我们调用返回的closure函数时，它仍然能够正确计算10 + 5 = 15。这就是闭包的神奇之处 ——内部函数inner记住了它定义时x的值是 10**。

2.2 自由变量与变量捕获机制

在闭包中，被内部函数引用但不是在其内部定义的变量称为自由变量。闭包的核心机制就是自由变量的捕获：内部函数会捕获并保存对这些自由变量的引用，即使外部函数已经执行完毕。

2.3 闭包与内存管理机制

理解闭包的内存管理机制对于掌握其原理至关重要。在 Python 中，每个对象都有一个引用计数（reference count），当引用计数为零时，对象就会被垃圾回收。

闭包的一个关键特性是它能够延长变量的生命周期，使其脱离调用栈（stack），进入堆（heap）语义。当外部函数执行完毕后，其局部变量通常应该被销毁，但由于闭包的存在，这些变量的引用计数会增加，因此不会被立即回收。

2.4 闭包与一般函数嵌套的区别

理解闭包与一般函数嵌套的区别很重要。所有闭包都是内部函数，但不是所有内部函数都是闭包。关键区别在于：

  • 一般函数嵌套：内部函数只能在外部函数执行期间访问外部变量，外部函数执行完毕后，内部函数也无法访问这些变量

  • 闭包：内部函数在外部函数执行完毕后，仍然能够访问并操作外部函数的变量

从本质上讲，闭包是函数与其执行环境的组合。更准确地说，闭包是内部函数与外部函数执行环境（包括其变量）绑定在一起的对象。

这种绑定机制通过 Python 的__closure__属性实现。当内部函数捕获自由变量时，Python 会创建特殊的cell 对象来保存这些变量的值。这些 cell 对象被存储在__closure__属性中，形成了一个持久的引用链。

2.5 闭包使用场景函数工厂（Function Factory）

①函数工厂

闭包最常见的应用之一是创建函数工厂，即动态生成具有特定行为的函数。这种模式在需要创建多个相似但参数不同的函数时特别有用。

场景描述：假设你需要创建多个不同基数的幂函数（如平方、立方、四次方等），传统方法需要为每个幂函数单独定义，而使用闭包可以优雅地解决这个问题。

在这个例子中，make_power是一个函数工厂，它接受一个参数exponent（指数），并返回一个闭包power。每个通过make_power创建的函数都 “记住” 了自己的指数值，因此可以正确计算相应的幂。

优势分析：

  • 代码复用度高：只需要定义一次power函数

  • 参数定制化：可以根据需要创建任意指数的幂函数

  • 代码简洁：避免了为每个幂函数单独编写代码

②状态保持器

闭包的另一个强大应用是实现状态保持，即在函数调用之间保存数据。这在创建计数器、累加器或需要记忆之前状态的函数时非常有用。

场景描述：创建一个简单的计数器，每次调用时返回递增的整数。传统方法可能需要使用全局变量，但全局变量容易被意外修改，而闭包提供了更好的解决方案。

在这个例子中，create_counter函数返回的counter函数形成了一个闭包。每次调用counter时，它都会增加count变量的值并返回。由于闭包的特性，count变量的值在多次调用之间得以保持。

关键特性：

  • 数据封装：count变量被封装在闭包内部，外部无法直接访问或修改
  • 独立性：通过create_counter创建的多个计数器之间完全独立，互不影响
  • 可配置性：可以通过initial_value参数设置计数器的初始值

五、装饰器

含义:
装饰器本质上是一个Python闭包函数，它可以让其他函数在不需要做任
何代码变动的前提下增加额外功能，装饰器的返回值也是一个函数
对象。
装饰器对于超长和复杂代码功能的添加十分有用，避免了直接修改代码中的函数功能，避免出错。

装饰器需要满足以下两点：
1.不修改原程序或函数的代码
2.不改变函数或程序调用方法。

1.1 标准装饰器（本质是一个闭包，但参数是一个函数）

装饰器的原理就是将原有的函数带入到一个闭包种，重新定义为以原有的函数为参数的闭包。

这个闭包的外层函数参数是原函数，即你想要装饰添加功能的函数。
装饰不同函数的时候都需要重新定义一个变量接收outer(function)。

1.2 语法糖装饰器

@outer 要顶格写，与函数间最好不要空行。且装饰器名称这里不用加上括号，不加是引用装饰器，加了就是调用函数。
