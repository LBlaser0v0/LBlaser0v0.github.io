5.方法的**重载**定义：

  • 同一个类中，定义了多个同名的方法，这些方法具有类似的功能；
  • 每个方法具有不同的参数类型和参数个数，这些同名的方法，就构成了重载关系；
  • 简单理解：同一个类，方法名相同，参数不同的方法，无需看返回值。

调用重载的函数其中一个时，会调用形参和实参一一对应的那个。

2.变量和方法的内存分配：

函数传参：

传入变量时，形参变量在函数内改变则不改变原先的实参变量；

但传入数组时，在函数内改变数组，会使函数外的数组也发生改变。

在java中对不同内存的调用：

①栈内存

每个线程都有自己独立的栈，目前只要考虑一个即可，方法被调用进栈执行，执行完毕出栈。

栈与方法有关。

比如main方法，作为程序的主入口，程序开始执行时会进栈，代码全部执行完时会出栈。

②堆内存：

所有线程共享，存储对象、数组、字符串常量池。

用到new这个关键字时，就会用到堆内存。

new关键字在内存中开辟了一块空间并附上了地址。

③方法区：

JDK7以前，方法区由永久代实现，JDK8+由元空间实现，从虚拟机内部移到本地内存。存储字节码信息、常量、静态变量。

程序编译完的字节码信息存入到方法区，即**.class字节码文件**会被加载到方法区临时存储，但不运行。

④本地方法栈：

调用本地Native方法，一般用不到这个知识。

⑤程序计数器：

每个线程独立、记录当前线程执行的字节码指令地址（行号）。平时也用不到。

重点记前三个。

基本数据类型不用到new，也就不用到堆内存，其中记录的是真实的数据，传递的也是真实的数据。

2.static关键字的应用

①static修饰成员变量：

当某一类中有某个属性总是一样的时候，用static修饰这个属性，使这个属性在每个类中都是一样的。

如一个班的学生总是有同样的老师。

static：表示静态，是Java的修饰符，用来修饰（成员变量/成员方法）

特点：叫做静态变量，被该类所有对象共享，公共的变量，不单属于某个对象，而属于类。

此时任意一个对象修改了这个成员变量，其他对象中的这个成员变量也会跟着改变。

静态变量调用方式：
方式一：类名调用（推荐这种写法，更规范）。
方式二：对象名调用

static关键字的内存解析：

如果对类中的静态变量进行赋值，会先将类文件存入到方法区，并未这个静态变量在内存静态区中开辟一个新的空间。

可见，静态变量是随着类的加载而在内存静态区中开辟空间的，与对象无关。

静态区的位置在jdk8前是在方法区，在jdk8后就存在于堆内存。

②static修饰成员方法

static关键字修饰成员方法
特点：
static修饰的方法叫做静态方法，该方法多用在测试类和工具类中
Javabean类中很少会用
调用方式：
方式一：类名调用（推荐）
方式二：对象名调用

非静态方法只能通过创建对象后调用。

工具类（新名词）：

不是用来描述一类事物的，也没有main方法，而是帮我们做一些事情的类。

很好的例子比如通常调用的System.out.println的方法，他就是从工具类中调用的，一个工具类中可以放我们自己写的通常需要用到的方法（函数），使他们能够在许多类中可以快速地复用，而不用在写一遍。

比如自己写一个求最大值/最小值，遍历数组的方法放到工具类中。

工具类的一些要求：

1.类名、方法名要见名知意；
2.私有化构造方法，不让外界创建对象，因为不需要。
3.所有方法都使用静态的（static），直接使用类名调用，而不是对象名调用。

③静态的一些注意事项：

  • 静态方法只能访问静态变量和其他的静态方法
  • 非静态方法可以访问静态变量或者静态方法，也可以访问非静态的成员变量和非静态的成员方法
  • 静态方法中是没有this关键字

总结：静态只能调用静态，非静态可以调用所有，静态方法无this

原因在于静态的变量或方法是随着类的创建一起出现的，先于非静态的事物和对象存在，与对象是否存在无关。

而非静态变量或方法随着对象的创建才出现，与对象有关。

现在可以重新认识以下main方法：

  • public: 被JVM调用，访问权限最大
  • static:被JVM调用，类名访问，测试类中其他方法也都要是静态的
  • void:不需要给JVM返回值
  • main:固定的名字，被JVM识别
  • String[]args:)用于接收运行的参数，目前不用

一、类的继承：

自己简单理解，就是子类会继承父类可访问的成员变量或者方法。比如，父类Person中有name、age等成员变量，那么子类Student一样可以访问这些成员变量，并创建对象。

准确来说，继承是类与类之间的一种父子关系，Java中提供关键字extends，用于建立类与类之间的关系。

继承的好处：

  • 可以把多个子类中重复的代码抽取到父类中了，提高代码的复用性
  • 子类可以在父类的基础上，增加其他的功能（新方法，新成员变量），使子类更强大
  • 子类不必重复写父类中的成员变量和方法，除非想重写。

提前记一下方法重写：当子类中的一个方法的名字，形参，返回值与父类中的对应方法完全相同时，就构成方法重写；此时子类调用这个同名方法时会优先使用自己的方法，如果不重写则优先使用父类方法。

二、继承结构的正确设计

什么时候用到继承：
具体来说：当类与类之间，存在相同（共性）的内容，并满足子类是父类中的一种，就可以考虑使用继承，来优化代码。

总的来说，类的继承既要满足子类有共性，并且要符合现实的事物分类情况。

技巧：

  • 使用类的继承之前，可以先画图，从子类往父类画，提取共性；
  • 编程类的继承时，先从父类写起，再写子类继承。

三、继承的特点

1.对于Java来说，继承只有单继承，不支持多继承，但可以多层继承。

通俗来说，一个子节点只有一个父节点，一个父节点可以有多个子节点，并且可以有爷爷节点（多层继承中称其为间接父类）。

继承的结构特点，可以近似归纳为树形结构。

2.java中有一个顶级父类Object，是所有被创建类的父类

  • 完整搜索链条为：先找子类中的成员方法—>直接父类的成员方法—>间接父类的。

成员方法的重写：

即：在子类中，把父类的方法再写一遍，方法申明保持一致，包括方法的名字、形参列表都要保持一致、子类重写后的方法返回值类型要是父类的子集（被父类返回值类型包含）。

**使用场景：**如果父类的方法不能满足子类的要求了，子类中可以把该方法再重写一遍。

快捷重写的方法：在子类中同时按下Alt+Insert，在弹框中选择Override Methods，即可查看可重写的父类方法。

@Override是一个给java虚拟机看的标记，标识下面是一个重写而来的方法。

成员方法重写的要注意：
  1. 如果父类里面的代码，我一行都不想用，此时把子类中的方法体重新完整写一遍即可；
  2. 如果父类里面的代码我还想用，此时我只是在父类的基础上再加其他的逻辑，那么此时可以先通过super关键字调用父类的方法得到一个结果，再对这个结果进行操作。
  3. **了解：**子类重写父类方法时，访问权限子类必须大于等于父类（访问权限权限：private<空着不写<protected<public）
  4. **了解：**子类重写父类方法时，返回值类型子类必须小于等于父类。
  5. 建议：重写的方法申明和父类保持一致即可
  6. final修饰类为最终类，里面所有的方法不能被重写
  7. 如果一个方法被private、static、final修饰，则它们都不能被重写。
3.对于构造方法

最关键的一点：与一般方法不同，子类不继承父类的构造方法，只能通过super关键字调用。

父类中的成员调用super构造，子类中的用this自己构造。

  • 继承结构当中构造方法的小细节：

  2. 如果想要访问父类的带参构造，super（参数）必须手动写上，不能省略
  3. 在创建对象的时候，先执行父类的构造方法，再执行子类的构造方法

讨论static修饰符：

static 成员属于类本身，而非对象。从“继承”的严格定义（子类自动获得父类的实例成员）来看：

  • 静态变量：子类可以访问（如果访问权限允许），但不是“继承”过来的副本，而是共享同一个类变量。子类也可以定义同名的静态变量，这叫隐藏，不是重写。
  • 静态方法：子类可以调用父类的静态方法（权限允许），也可以定义同名的静态方法，同样只是隐藏，不会发生多态。

结论：静态成员可以被继承（访问），但不参与多态，且本质上是父类的成员，不是子类独有的。

五、继承中的一些底层特点

讨论继承过程中内存的结构，以及.class字节码文件详解。

1.继承的底层原理：

需要分清楚继承和能够调用的区别，子类的继承是复制了一份父类的成员给自己用，但有时父类的成员子类也能调用，但子类不拥有它，不能说继承了这个成员。

2.子类真正能继承父类的东西是什么：

构造方法：不能被子类继承，可以利用super关键字调用
成员变量：可以被子类继承，private私有的也可以，但是私有的无法直接调用

成员方法：

  • 虚方法可以被继承（虚方法就是不加下面三种修饰的方法）；

  • final修饰的最终方法不能被继承，可以被调用，编译期确定位置，运行时直接调用；

  • static修饰的静态方法不能被继承，可以被调用，编译期修改为类名调用，运行时直接调用；

  • private修饰的私有方法不能被继承，不能被调用；

  • 方法重写：子类替换虚方法表里面方法的地址值。

总结一下就是：构造方法不继承；成员变量都继承，但private修饰的特殊；成员方法中需方法都可以继承和调用，被final、static修饰的调用性质相同，private修饰的最严格。

final和static的成员方法调用特点：

final修饰的最终方法不能被继承，可以被调用

  • 编译期：一级一级往上找，遍历整个继承链，确定要调用的方法在A类里面，记录该方法的地址

  • 运行期：直接运行编译时确定的方法，虽然降低了编译的速度，但是运行的速度会得到保证

static修饰的最终方法不能被继承，可以被调用

  • 编译期：一级一级往上找，遍历整个继承链，确定要调用的方法在A里面将对象调用方法直接修改为类名调用方法
  • 运行期：直接运行

所以我们可以看出，调用static修饰的方法时，最好类名调用，而不是对象调用，可以省去编译期的查找流程，节省时间。

3.继承结构的内存图解：

图例解释：继承结构在内存分配上有两点与之前不同。

  • ①在方法区中，示例代码会先载入父类的字节码文件再载入子类的字节码文件；
  • ②堆内存中，存放子类成员的空间与父类的会有所区分，图示代码会先访问子类成员所在的堆空间，找不到则访问父类方法的堆空间。

private修饰的成员变量，可以被继承，但只能通过get/set进行调用。

static静态变量放到静态区，而不在堆内存中。
