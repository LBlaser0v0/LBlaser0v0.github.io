一、多态

1.什么是多态

顾名思义，多态就是事物多种形态。多态是为了服务相似但有微小区别的接口的。

子类对象在传参时可以赋值给父类类型实现多态，换种说法，方法的形参写父类类型，则就可以接收所有子类参数。

如：一个给学校人员程序注册的场景

使用多态可以将多个子类的同一个要利用的方法整合起来，利用传递父类形参进行子类的方法实现，节省代码，易于复用。

2.多态的表现形式

父类类型 对象名称 = 子类对象，这种形式也可以扩展为上面说的函数传参。

3.多态的前提（记住）

多态的前提：

  1. 有继承/实现关系必须的
  2. 有父类引用指向子类对象必须的
  3. 有方法重写可选的

4.多态的好处：

  • 方法中使用父类型作为参数，可接收父类对象+所有子类对象
  • 如果进行方法重写，利用多态调用方法，可以调用不同子类中重写的方法

成员包括成员变量和成员方法。

6.多态的弊端与解决（重点）

从上面的分析可以看出，成员方法是实现多态的关键。

类的类型转换：

  • 自动类型转换(从子到父)：子类对象赋值给父类类型的变量
  • 强制类型转换(从父到子)：子类 对象变量 = (子类)父类类型的变量

利用类型转换解决多态的弊端。，把对象再转回子类类型即可。

二、抽象类和抽象方法

1.什么是抽象类和抽象方法

抽象类的实现，能够增强代码的规范性，使得父类称为抽象类，子类则必须重写父类的抽象方法，而父类不能创建对象。

抽象类和方法的前缀：abstract

抽象方法：在父类中不能确定具体的方法体。该方法就可以定义为抽象方法。

**抽象类：**如果一个类中存在抽象方法，那么该类就必须声明为抽象类。

定义格式：

抽象类和抽象方法的关系：有抽象方法的类必须是抽象类。

作用：

  • 抽象类声明一个通用的模板结构类，父类不用再纠结子类写了什么；
  • 让子类必须重写抽象方法，实现结构统一，也不用重复定义同样的方法。

2.注意事项

注意点1：抽象类不能实例化
注意点2：抽象类中不一定有抽象方法，有抽象方法的类一定是抽象类
注意点3：抽象类中可以有构造方法，为了给子类继承的成员变量赋值。
注意点4：抽象类的子类，要么重写抽象类中的所有抽象方法，要么子类也是抽象类。

由点2可以引申出，抽象类可以不写抽象方法，如果单纯不想一个类被实例化创建，那么可以再前面加上abstract。

3.意义

在企业级的项目开发中，写抽象类是很有必要的，因为不同人写的同一种功能方法的模式很可能不同，导致在开发过程中增加了沟通成本和代码阅读难度。

所以，抽象类的关键意义就是定义规范书写方法，强制同一功能的方法格式相同，方便协调管理。

三、接口

1.什么是接口

接口相当于人为额外定义的一个规则，如果某一个类想要实现这个规则，那么更这个接口关联即可。

当某一种方法写到父类不合适，但是又有多个子类要重用时就可以定义方法给这些子类使用。

简单说

接口：接口就是一个规则，而且是独立于继承体系以外的规则。（可以理解为干爹）

2.接口的定义

格式：利用关键字interface定义。

使用：接口与类之间称为实现关系，以关键字implements来表示。

创建文件时记得不选class，要选择interface。

类实现接口：

implements的使用类似extends实现类继承的方式。

3.注意事项：

  • 注意点1：接口不能实例化
  • 注意点2：接口的子类（实现类），要么重写接口中所有的抽象方法，要么实现类是一个抽象类
  • 注意点3：一个类可以实现多个接口，也可以在继承一个类的同时，实现多个接口（父亲只有一个，干爹可以有多个）

接口中的方法都是抽象方法，也不能创建实例。

类继承父类和多个接口演示：

注意：多个接口中有同名方法时，继承的类只用重写一次即可（如果继承的抽象父类也有这个方法则也只用写一次）。

4.接口中成员的特点：

**成员变量：**接口中成员变量只能是常量，不可改，默认修饰符是public static final。

  • 接口中成员变量必须手动赋值。

构造方法：接口中不存在。

**成员方法：**只能为抽象方法，默认修饰符是public abstract。JDK7及以前的版本不写abstract会直接报错，更新的版本不写也会默认加上abstract。

接口与接口之间的关系：可以多继承，也可以单继承

注意点：
1．如果有一个接口A继承了多个接口，此时相当于是把多个接口中的抽象方法全部继承下来了
在以后，实现类实现接口A的时候，就要把所有的抽象方法进行重写

5.接口在新版本的特性

  • JDK7及以前：接口中只能定义抽象方法

  • JDK8新特性：为了接口升级时向下兼容，接口中可以定义有方法体的方法(默认、静态)

  • JDK9新特性：为了抽取重复代码，接口中可以定义私有方法(私有普通方法，私有静态方法)

java加入新特性的目的：
  • 当一个接口在持续开发过程中需要增加功能时，先前版本不得不对继承这个接口的子类也进行相应的方法重写，否则会报错。
  • 为了解决这个问题，java后续就开放了在接口中可以写有方法体的私有方法，这个方法不强制要求子类马上重写，而是在要用到这个方法的时候再进行重写。
JDK8接口新增定义默认方法（default）：

作用：接口升级代码不报错。

接口中默认方法的定义格式：

  • 格式：public default 返回值类型方法名(参数列表){}
  • 范例： public default void show(){}

接口中默认方法的注意事项：

  • 默认方法不是抽象方法，所以不强制被重写。但是如果被重写，重写的时候去掉defaut关键字
  • public可以省略，default不能省略
  • 如果实现了多个接口，多个接口中存在相同名字的默认方法，子类就必须对该方法进行重写
JDK8接口新增定义静态方法（static）：

作用也是为了升级接口

接口中静态方法的定义格式：

  • 格式：public static 返回值类型方法名(参数列表){}
  • 范例： public static void show() { }

接口中静态方法的注意事项：

  • 静态方法只能通过接口名调用，不能通过实现类名或者对象名调用（这里类似类中的静态方法，可以用类名+方法名直接调用，不用创建实例）
    但与之前也有不同，父类中的静态方法可以用子类的对象名+方法名调用，接口的静态方法则只能够用接口名调用。

    上图aaa方法在父类中，创建子类调用不报错，而method方法在接口（干爹)中，子类调用会报错。

  • public可以省略，static不能省略

  • 静态方法写到子类中时，不能加@Override这个注解，因为静态方法不能被写到虚方法表中。（private、static、final都不行，只有虚方法表里的方法才能被重写）。

JDK9接口新增定义私有方法（private）：

作用：私有方法是为了抽取默认方法和静态方法的相同方法体部分而出现的，它不需要被外界调用和重写。

**相当于一个只为接口内部服务的功能模块，增加接口方法书写的结构强度。**不能被调用也保护了接口内部代码的安全性（JDK8时，使用default来实现这个功能，但default能直接被外界调用，不够安全）。

格式：(一般私有和静态私有)

  • 格式1：private 返回值类型方法名（参数列表）{}
  • 范例1：private void show(){}
  • 格式2:private static 返回值类型方法名（参数列表）{}
  • 范例2:private static void method(){}

格式1是当多个default方法有共同部分时来提取相同代码功能的；
格式2是当多个static方法有共同部分来提取相同代码功能的。

4.静态内部类：

5.局部内部类

（1）什么是局部内部类：

即在方法中创建的内部类。

1．将内部类定义在方法里面就叫做局部内部类，类似于方法里面的局部变量。
2．外界是无法直接使用，需要在方法内部创建对象并使用。
3．该类可以直接访问外部类的成员，也可以访问方法内的局部变量。

注意：局部内部类创建时，前面不能加public、static、private。

6.匿名内部类(重要知识)

（1）什么是匿名内部类

顾名思义，就是一个没有名字的内部类

作用：继承一个类或者实现一个接口的时候少写一个java文件，也是lambda语句的前置知识点

定义格式：

匿名内部类的定义格式 = 没有名字的java类 + 继承/实现 + 重写方法 + 创建对象。
最终格式可以理解为：是一个没有名字的java类的对象
