Unity学习记录————应用基础

Unity学习记录——阿发跟学

一、基础

1.场景

物体的移动：红色的x轴，绿色的y轴（向上），蓝色的z轴

物体GameObject的创建：Cube立方体，Sphere球体，

gzimo导航器：在Scene3D界面的右上角，指示我们现在的视图状态和实际位置。

栅格Grid：表示XZ轴形成的坐标平面，position y的值为0。

天空盒Skybox：表示出游戏世界的背景。

视图相关操作：

1.旋转视图：Alt + 左键，在scene中旋转视角。
2.缩放视图:鼠标滚轮或ALT+RMB，精细缩放。
3.平移视图:MMB 鼠标中键。

导航器相关操作：

1.按 Shift，点击中间的小方块，恢复方向
2.点Y轴，顶视图
3.点X轴，右视图
4.点Z轴，前视图

视图中心点：

unity中视野的旋转是围绕着事业中心进行的，无法直接围绕着指定物体进行旋转视角。

想要视野围绕着一个物体旋转，首先要将该物体放到视野中心。

**方法：**1.框显所选，选中一个物体
2.按F键，置于视图中心（或者在层级管理器中双击这个事物）
3.此时旋转视图，似是绕着物体旋转，这里要使用Alt + 左键旋转。

出生点：添加一个新物体，物体位于视图中心
注意，出生点不是在(0,0,0)，而是在当前视图中心

透视与正交：

unity中有两种视图：

1.透视视图：Perspective，近大远小 ；
2.正交视图 Orthographic，又称等距视图 Isometric ；
透视图下，物体‘近大远小’。正交视图下，物体的显示与距离无关。

点击导航器下的字体，就可以在透视视图和正交视图之间切换。

正交视图主要用于对其物体，消除视觉误差。

2. 3D原生物体

Unity下自带的几种基本模型:
Cube，立方体； 边长1单位
Sphere，球体； 直径1单位
Capsule，胶囊体； 直径1单位，高2单位
Cylinder ，圆柱体；直径1单位，高2单位
Plane，平面。 边长10单位
Unity 尺寸单位，约定为1 米（但实际上是根据想创建的对象之间的相对大小来确定的）。

原生模型仅用于练习，真正的模型应在建模软件中制作
CG建模软件：3Dmax ／ Maya ／ Blender ／ Cinema4D/ ZBrush。

物体的操作：

移动、旋转、缩放，使用Scene左侧的工具

多选：

先选中一个物体（Scene中，或者层级管理器中（推荐）），再按住ctrl或shift进行多选操作。

复制duplicate：

选中一个物体，快捷键Ctrl+D，复制出一个新物体。

激活active：

控制物体的显示/隐藏，当暂时不需要这个事物，但又不想删除它时。

可以在检查器中通过取消勾选隐藏他。

对齐：

通过进入正交视图，调整x，y，z轴方向上的各个物体的位置进行对齐。

Move Tool 移动工具

点击箭头可以仅朝对应的方向移动，点击平面可以仅在改平面上进行移动。

Rotate Tool，旋转工具
演示：选中一个物体，绕Z轴旋转。
其中,可以在Transform中的rotation项里看到绕着各个轴方向的旋转角度。
Unity规定逆时为正，顺时针为负
可以在 Inspector 里精确指定
按住 CTRL 键后再去旋转时，角度增量为 15 度
Edit|Grid and Snap Settings 栅格吸附设置

Scale Tool ，缩放工具

可以选中一个物体缩放，分为轴向缩放，整体缩放。

我们可以通过QWER来对工具进行切换。

3.模型：

模型都是由若干个面围成的，模型内部不需要被看见，所以一般什么也没有。

①网格Mesh：

网格存储了模型的数据。

  • 模型的形状，由若干个小面围合而成
  • 模型是中空的
  • Mesh中包含了面、顶点坐标、面的法向等数据

在 Unity 中观察模型的网格

  • Shaded 着色模式，显示表面材质
  • Wireframe 线框模式，仅显示网格
  • Shaded&Wireframe线框着色模式

显然，任何物体的表面都是由若干三角面围成（三角面可以被视为渲染面数的最小单位），面数越多，物体表面就越精细，GPU负担也越重。

高模：指的是面数极多，精细度很高的模型。

②材质Material：

定义了物体的表面细节，如：颜色，金属／非金属，光滑／粗糙，透明／半透明／不透明，凹陷／凸起。

**给物体添加一个材质：**材质是一种资源，在assets目录下保存和管理，脚本等其他资源也在此管理。

  • 项目管理框中：在assets下创建一个资源管理文件夹Materials——>在该文件夹下右击找到create，进入后点击material。
  • 在创建好的材质中可以修改它的参数，如：物体表面颜色（反照率albedo），透明度，金属度（metallic），光滑度（smoothness）等。
  • 最后将这个创建好的材质拖到这个物体，即可使用。
③纹理Texture：

也称为贴图，用一张图片去表示物体表面颜色。比如往建好的模型上贴上设计好的图片。

建模贴图属于建模师的工作，当我们拿到别人做好模型的时候，贴图已经被贴好了。

④网格、材质、纹理之间的关系：
  • 网格规定了一个物体的基本形状，点、线、面这种特殊特点；
  • 材质定义了物体的颜色，粗糙度，透明度等外表属性，定义光学特性；
  • 纹理为物体贴上了贴图，相当于给物体穿了一层衣服，更细节的定义了物体表面不同位置的颜色，定义表面像素颜色。
⑤外部模型的导入（FBX）：

标准模型的文件格式是：FBX（也是Unity支持的格式）
fbx文件下一般包含了模型的网格和材质信息，贴图一般独立在外。

可以通过各种CG建模软件：3Dmax/Maya/Blender/Cinema4D/ZBrush，在建模软件中，将模型导出为FBX格式即可。对于外部导入的模型，unity通常以亮蓝色显示。

FBX的使用方式：

导入Unity时，直接将fbx文件和贴图拖入到相应的文件夹中即可，可以在Unity中的项目文件夹下操作。

对于一个FBX文件的贴图：

  • 有的模型需要贴图文件
  • 贴图文件的路径是约定好的
  • 放在与fbx相同目录，或者同级的Textures目录中

同时一个物体也可能在不同表面有不同的材质，那么我们可以在检查器中找到Mesh网格中的Materials项，为这个物体设置多个材质。材质一般是放在这个物体下的，或者说挂载到这个物体上的。

材质替换（材质重映射Remap）：

对于已经给定好的物体材质，有时候是无法修改里面的参数的（只读文件）。
那么可以选中*.fbx资源文件，在 Inspector 中切到JMaterialss 属性，Use Embeded Materials，然后On Demand Remap：映射新的材质，点Apply应用设置。

使用外部材质：

  1. 选中 fbx 文件，在Materials的上方。
  2. 选中Location：改为Use External Materials 使用外部材质
  3. 点 Apply 应用，则将内嵌材质解压缩到 Materials 目录
  4. 直接修改 Materials 目录下的材质文件

这样操作之后，会在模型同目录下生成一个新的材质，原本嵌入fbx文件的不可修改材质，就可以用新材质替换。

4.物体细节：

平面：所有的平面都是没有厚度的，并且只渲染正面，背面透明。这样做是基于一般物体都是从外表面观察的，没有内表面观察的需要。

6.项目资源：

6.1资源文件目录Assets。

常见类型：

  1. 模型文件 Model*.fbx
  2. 图片文件Texture（*jpg / png / psd / tif )
  3. 音频文件 AudioClip(*.mp3 / wav / aiff)
  4. 脚本文件 Script (*.CS)
  5. 材质文件 *.mat，场景文件 *.unity。

Assets采用目录式管理，和windows管理逻辑非常类似。

几个操作：

  • 在文件夹窗口查看，Show In Explorer
  • 添加、删除目录
  • 添加资源文件，文件会被拷贝到项目中
  • 复制资源，CTRL+D
  • 缩略图比例调节、列表模式
6.2场景文件夹Scenes：

场景文件就是scenes文件夹下的.unity后缀的文件，它们储存了一个游戏场景的信息。

一个场景代表一个关卡，可以创建多个场景
几个操作：

  • 添加场景，creat + scene即可
  • 打开场景
  • 编辑场景，保存 CTRL+,S
  • 在入门教程中，只使用一个场景。
6.3 资源包UnityPackage

资源包，即是对Assets资源文件夹进行打包。打包是为了更好的转移或者共享资源。

导出资源包步骤：

  • 选择一个资源文件，或者一个目录
  • 右键，Export Package
  • 生成*.unitypackage资源包

资源包的导出对象可以是一个单一资源、一个目录下的资源或者整个项目。

导入资源包更为简单，直接将.unitypackage文件拖到项目管理窗口即可。

注意：实际项目开发中，如果导入的场景包包含Scenes文件夹，记得要改名字，不然导入的项目目录结构会乱。

6.4 Unity官方资源商店：

资源商店 Asset Store ，https://assetstore.unity.com
也可以在 Unity 打开，Window丨Asset Store
有一些免费或收费的资源，包含模型、材质纹理等。

7.物体操作和理论了解：

7.1轴心pivot：

轴心是一个物体操作的基准点，对于一般物体，它的轴心就位于集合中心位置。

移动，旋转等一些基础操作都是围绕着这个点（点上的轴）进行的。

但是对于自建模的物体，轴心可以定义在任何位置，比如一个圆环，其轴心并不在物体之上。

unity中改不了轴心点，只能在建模软件中修改。

7.2父子关系：

父节点与子节点的关系，一些物体可以关联在一起形成一个整体功能，分为父子关系可以帮助管理。

即如果两个物体模型有着从属关系，就可以将子物体挂到父物体之下，从而控制父物体时子物体也能一起管理。

简单来说，父子级，指两个物体之间的关系

  • 在Hierarchy窗口中，
  • 拖动物体B，到 物体A下
  • 子物体会随着父物体一并移动
  • 删除父物体时，子物体一并删除
  • 比较特殊的一点是，子物体的坐标是相对于父物体的轴心坐标系定义的，而非世界中心坐标系。
7.3空物体Empty：

Creat Empty创建一个空物体。空物体是不可见的，没有网格参数。但有坐标，可以移动操作。空物体的作用类似于一个接口、节点或者说容器。

空物体其实作用很大：

  1. 它可以作为一个父节点，用于节点的组织和管理
  2. 可以用于标记一个位置

比如，我们可以将两个相关物体一并挂载在一个空物体上，管理这个空物体就可以一起管理这两个物体。

**同时如果我们向修改一个物体的轴心时，也可以设置一个空物体，将物体挂载在空物体上，那么物体可以通过这个空物体的轴心进行操作。**用父节点空物体轴心代替子物体的轴心，变相修改了一个物体的轴心。

7.4 Global与local坐标系：

1 Global，即世界坐标系

  • 以世界中心为轴
  • 6个方向代表：上下东西 南北

2 Local，即本地坐标系

  • 以物体自身为轴
  • 6个方向代表：上下前后左右

对物体的操作时，可以选择两个坐标轴其一。

一般物体旋转时，**Local坐标系z轴始终朝物体最初设定的前方，而Global坐标系不会随着改变。**这是编写物理移动的脚本逻辑的重要理解前提，物体的移动一般是沿着Local来移动的。因为前后左右移动是在Local坐标系下才有的定义。

**注意：有个模型通常方便于开发的设定，即y轴称为 up，z轴称为 forward，×轴称为 right，一般地，要求模型的正脸与Z轴方向一致。**所以forward z轴对写代码来说最重要。

7.5 pivot轴心和center几何中心模式：

可以在scene窗口控制栏进行轴心模式和几何中心模式的切换对物体的操作模式，通常情况下基本使用的是pivot轴心模式。

一般来说，物体的轴心和集合中心的位置不同。

简单理解，pivot轴心是物体建模时定义的物体支点，center几何中心是物体本身的中心。

有一个比较明显的区别就是，选中多个物体的情况下进行旋转，pivot模式下，各个物体绕着各自的轴旋转；center模式下，物体会绕着多个物体结合后的几何中心旋转。

8.组件component：

8.1什么是组件：

每个组件代表一个特定功能，需要什么功能，就添加什么组件。

在inspector检查器中带有各个物体上不同的组件。

组件例子：

  • 光源，Light；
  • 网格过滤器，Mesh Filter：用于加载物体的形状，包含着点线面、法向等集合数据。
    网格过滤器的内容可以通过FBX等建模文件给出，使网格过滤器的内容关联建模文件的网格信息。
  • 网格渲染器，Mesh Renderer：用于将网格过滤器中加载的形状渲染出来。

总结网格过滤器用于存储物体信息，网格渲染器用于渲染出物体的形状。

想要物体实现什么功能，就要添加什么样的组件（Add Component）。

Add Component在Inspector检查器中，可以进行组件添加。

组件的删除也很简单。

8.2 AudioSource组件：

这个组件用于播放音乐和音效。

使用方式：

  • 准备一个音频文件，.mp3/wav/aiff等
  • 创建一个空物体
  • Add Component ，在Audio列表项中找到Audio Source
  • 将音乐文件拖到 AudioSource.AudioClip 属性
  • 在3D窗口上方，选Toggle Audio On
8.3 transform组件：

transform组件被称为变换组件。

包含着物体的基本参数：

  • Position，物体位置（与父物体的相对坐标）
  • Rotation，旋转(欧拉角)
  • scale，大小缩放

特点：所有物体都有transform组件，不可被删除。

transform组件在脚本中经常被使用。

8.4 摄像机：

摄像机Camera，负责拍摄游戏画面。就像一个演出拍摄的摄影师。
解释：

  1. 调整窗口布局，Scene／Game 并排显示
  2. 选中摄像机，观察镜头范围
  3. 摄像机的Z轴指向，即为拍摄方向
  4. 拍到的画面，呈现在 Game 窗口中
  5. Game窗口中的画面，即为玩家最终所见画面

调整摄像机的角度，两种方法:

1.手工移动、旋转摄像机，对准目标

2.Align with View，与 3D视图对齐

  • 先在 3D视图里摆好角度，此为观察者视角
  • 然后选中 Main Camera，执行 Align with View
  • 此时，摄像机视角与观察者视角完全相同
