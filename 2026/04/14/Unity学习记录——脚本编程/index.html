Unity学习记录————脚本编程

Unity学习记录——阿发跟学

一、脚本基础

1.什么是脚本：

script脚本，用于驱动游戏逻辑。

Unity中创建的脚本文件名就是类名，不可写中文，按照类名的命名规范书写。

注意C#脚本一般不能离开一个项目运行，所以我们要把脚本挂载的一个物体上去测试它（可以是一个空物体）。

2.脚本编写基本流程：

  1. 创建脚本文件；

  2. 分类管理脚本文件；

  3. 撰写脚本功能；

  4. 保存脚本；

  5. Unity自动编译脚本；

  6. 挂载脚本（到物体），有两种办法：

    ~点Add Component，选 Scripts | Simple Logic；
    ~或者，直接将脚本拖到小Inspector窗口的最下方。

  7. 运行游戏测试脚本实现。

    ~在 Console 窗口中，观察输出。
    ~若Console 窗口未显示，则Window丨General丨Console 调出此窗口。
    ~最后停止游戏即可

4.Unity上编写脚本的要点（重要）

  1. C#文件名必须和类名一致，否则无法挂载（固定要求）
  2. 若要修改类名，应在VS 中右键重命名
  3. 脚本必须挂载到物体上，才会被调用（脚本要挂载才会被创建对象，被实例化）
  4. 其他问题，参考《Unity编译环境配置》

第1点可能是由于Unity是采用反射技术来实现脚本的，并且实时动态的从文件加载类，而不是编译进程序。

总的来说，c#是面向对象的编程语言，脚本的编写大多在于对类和方法的创建以及调用

二、具体脚本编程情景：

1.编辑模式与播放模式

EditMode，编辑模式
Play Mode，播放／运行模式

在播放模式下：

  • 实时显示场景中的物体和属性
  • 游戏中的状态不可保存，You must exit play mode to save the scene

2.事件函数

（1）帧更新（Frame）

Frame，一个游戏帧
FrameRate，帧率／刷新率
FPS，Frames Per Sencond，每秒更新多少帧

帧率观察：

  • Time.time_，游戏时间（计算从play开始以及经过的世界）
  • Time.deltaTime，距上次更新的时间差

显然，帧率是不固定的，Unity 会尽量较快地更新

Unity 不支持固定帧率，但可以设定一个近似帧率
Application.targetFrameRate = 60;
其中，指示 Unity 尽量以 FPS = 60 的帧率更新游戏

帧更新函数：

Unity的C#编程中，我们利用Update()和LaterUpdate()进行与帧更新有关的操作。

其他的Awake()、Start()、等默认函数与帧更新无关。

Update：当MonoBehaviour启用时,其Update在每一帧被调用；

LateUpdate：当Behaviour启用时，其LateUpdate在每一帧被调用

FixedUpdate:这个函数会在每个固定的物理时间片被调用一次.这是放置游戏基本物理行为代码的地方。UPDATE之后调用。

（2）启动函数

①Awake()：

​ 在MonoBehavior创建后就立刻调用，在脚本实例的整个生命周期中，Awake函数仅执行一次；如果游戏对象（即gameObject）的初始状态为关闭状态，那么运行程序，Awake函数不会执行；如果游戏对象的初始状态为开启状态，那么Awake函数会执行；

​ 值得注意的一点是，**Awake函数的执行与否与脚本实例的状态（启用或禁用）并没有关系，而是与脚本实例所绑定的游戏对象的开关状态有关。**如果重新加载场景，那么场景内Awake函数的执行情况重新遵循上述两点。

②Start()：

​ 将在MonoBehavior创建后在该帧Update()第一次执行前被调用；Start()函数只在脚本实例被启用时才会执行；Start函数总是在Awake函数之后执行。如果游戏对象开启了，对象上绑定的脚本实例被禁用了，那么Start函数不会执行。

​ 这是Start函数的特点，只有在脚本实例被启用时它才会执行，并且Start函数只会在脚本实例首次被开启时才会执行。如果是已经开启过的脚本实例被关闭后再次开启，那么Start函数不会再次执行。

一般开发中都是在Awake函数中获取游戏对象或者脚本实例的信息，然后在Start函数中进行一些获取之后的初始化设置。

③一些调用顺序结论：

Awake、OnEnable、Start,都是游戏开始运行前就调用的方法。

GameObject的Activity为true，脚本的enable为true时，其先后顺序为：Awake、OnEnable、Start；
GameObject的Activity为true，脚本的enable为false时,只运行Awake；

GameObject的Activity为false时，以上都不调用，OnDisable()被调用；

OnDestory:物体被删除时调用。

比如,

  • 第1次，deltaTime = 0.016 秒，运动 0.01 米
  • 第2次，deltaTime = 0.030 秒，运动 0.01 米

所以计算物体的速度最好用deltaTime参与计算。

（5）物体的旋转调整运动方向

①简单更改物体朝向：

1.获取目标物体
GameObject flag = GameObject.Find(“物体名称”);
2.先转向目标
this.transform.LookAt(flag.transform);
3.再向‘前’运动，forward，+Z方向，Z为面朝方向。
this.transform.Translate(O, O, dz , Space.Self)

LookAt函数，指示脚本挂载的物体方向朝向另一个物体flag。使用LookAt的时候，要随时注意到两个物体是否都在地面上，否则跟随物体移动时会原地起飞。

GameObject.Find方法简介：

通过名字或路径查找游戏对象。

  • GameObject.Find(“GameObject”);

  • GameObject.Find(“GameObject/ChildGameObject);

使用规范:

  1. 无法查找隐藏对象
    隐藏对象包括查找路径的任何一个父节点隐藏(active=false)

  2. 如果查找不在最上层，建议合理使用路径查找，路径查找是把双刃剑

    **优点1:**解决查找中可能出现的重名问题。
    **优点2:**如果有完全的路径，减少查找范围，减少查找时间。

    缺点: 路径或结构调整后，容易影响到程序中的查找，需要重新定位查找路径。

  3. 如果路径查找中的任何一个父节点active=false，这个对象都将查找不到。

三、Unity脚本运行逻辑

1.脚本运行

点击play进行场景运行时，场景的加载过程示例：

  1. 创建节点

    GameObject node = new GameObject()

  2. 实例化组件

    MeshRenderer comp = new MeshRenderer()

  3. 实例化脚本组件

    SimpleLogic script1 = new 脚本类名()

  4. 调用事件函数

    • 初始化 script1.Start()
    • 帧更新 script1.Update()

实际上Unity引擎，会创建所有已激活的物体的各组件的实例，方便程序调用。

注意，Unity 是一个纯面向对象的框架，对象由框架创建，我们只需要配置好脚本和物体关系等细节即可，

2.消息函数：

（1）总体介绍

也称事件函数，前面在二、2的事件函数中有讲到一些。

消息函数有很多，这里记录一些常用的，其他的有待继续学习。

Start()：

  • Unity在运行开始时第一帧进行调用（第一次执行Update之前），比Awake()函数在MonoBehavior基类创建时就调用要更晚一些。

Update()：

  • MonoBehavior启用后，每帧都调用Update()。

已禁用的组件，其消息函数 Start／Update 不会被调用。
Awake／Start，都只会执行一次。

常见物种消息函数的简单理解：

  1. Awake，初始化，仅执行一次，即使脚本组件被禁用。
  2. Start，初始化，仅执行一次。
  3. Update，帧更新，每帧调用一次。
  4. OnEnable，每当组件启用时调用。
  5. OnDisable，每当组件禁用时调用。

（2）脚本执行顺序：

消息函数的调用顺序：

  1. 第1阶段初始化，

    script1.Awake(), script2.Awake(), …

  2. 第2阶段初始化，

    script1.Start() , script2.Start(), …

  3. 帧更新,

    script1.Update(), script2.Update()，…

默认地，所有脚本的执行优先级为0。
优先级的设定：

  1. 选中一个脚本，打开 Execution Order 对话框。
  2. 点+按钮，添加一个脚本。
  3. 指定优先级，值越小、优先级越高。或者，直接拖动调节顺序。

脚本的执行顺序，和Hierarchy 中的层级顺序无关。一般地，并不需要显式设置Execution Order，默认即可。

3.主控脚本

即一个游戏的主控逻辑，类似于一个程序的main函数。但复杂游戏的主控脚本不一定只有一个。

主控脚本中一般放着全局性的设计和逻辑，比如设置全局渲染帧率Application.targetFrameRate。

我们可以把主控脚本挂载到一个空物体上，只要这个主控脚本不实现具体的游戏操作功能。

（3）参数的赋值：（理解记忆）

参数的值，取决于最后那个对他赋值的。

赋值的先后顺序分为：

类成员直接赋值（或默认值）—>Unity组件中修改（如果可以）—>Awake函数中赋值—>Start()函数中赋值—>Update()函数持续更新赋值。

这些赋值不一定都有，但谁是最后就用谁。

（4）参数的类型

参数的类型，分为值类型、引用类型

  • 值类型：如 int，float，bool
  • 值类型（struct种类）：如Vector3，Color
  • 引用类型（class）：如String，GameObject，Transform，MeshRenderer

不同类型初始值一般不同。

变量参数也可以是引用类型，包括：

  • 节点，GameObject
  • 组件，如 Transform、MeshRenderer、AudioSource
  • 资源，如 Material、Texture、AudioClip 、
  • 数组类型

注意：引用类型在使用前一定要赋值，有所指向，否则会报错。

一些细节：

（5）Unity运行的调试：

在unity运行游戏时，我们可以实时的修改游戏的运行状态：

  • 我们可以在Scene3D界面拖动物体，实时修改。
  • 可以在Inspector检查器中实时修改每个物体实例的参数，或者调整脚本中的变量值**（无论是值类型还是引用类型都可以）**。

调试技巧：

这大大方便了我们对游戏的各项数值进行一个调试，测试最合适的数值。

但游戏过程中是不能保存我们调试修改后数值的，想要将一个组件调试后的状态保持下来，我们利用以下方法：

  • 在 Play Mode （运行游戏模式）下，我们可以先暂停游戏，在目标组件右上角三个点中找到Copy Component，赋值当前组件状态信息。
  • 在 Edit Mode （编辑模式）下，目标组件右上角三个点中找到Paste Component Values，将先前的组件信息粘贴到可保存的状态下来。
