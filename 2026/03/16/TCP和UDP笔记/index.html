TCP和UDP笔记

一、TCP 和 UDP 最核心区别（必背）

表格

对比项 TCP UDP
连接性 面向连接 无连接
可靠性 可靠（不丢、不乱、不重） 不可靠（可能丢、乱序）
传输效率 慢 快
头部开销 20~60 字节 8 字节
流量控制 有 无
拥塞控制 有 无
数据形式 字节流 数据报
使用场景 文件、网页、邮件 视频、直播、游戏、DNS

一句话记忆：

TCP 可靠但慢，UDP 快但不可靠。

二、TCP 详细知识点（最重要）

1. TCP 是什么？

传输控制协议

  • 面向连接
  • 可靠传输
  • 全双工通信
  • 基于字节流

2. TCP 三次握手（建立连接）

目的：确保双方收发能力都正常。

过程：

  1. 客户端 → SYN

    我可以连接吗？

  2. 服务端 → SYN+ACK

    可以，你能收到吗？

  3. 客户端 → ACK

    收到，连接建立！

经典面试题：为什么要三次握手，不是两次？

→ 防止已失效的连接请求又传到服务器，导致错误建立连接。

3. TCP 四次挥手（断开连接）

因为是全双工，双方都要关闭。

  1. 客户端 → FIN：我发完了
  2. 服务端 → ACK：收到
  3. 服务端 → FIN：我也发完了
  4. 客户端 → ACK：收到，关闭

TIME_WAIT 状态：

客户端最后等待 2MSL，确保对方收到 ACK。

4. TCP 如何保证可靠传输？

  1. 序列号 + 确认应答（ACK）
  2. 超时重传
  3. 流量控制（滑动窗口）
  4. 拥塞控制
  5. 去重
  6. 按序交付

5. TCP 流量控制

让发送方速度匹配接收方能力，防止接收方缓冲区溢出。

依靠 ** 滑动窗口（Window）** 实现。

6. TCP 拥塞控制

防止发送方把网络挤爆。

四个算法：

  1. 慢启动
  2. 拥塞避免
  3. 快重传
  4. 快恢复

7. 你刚才问的重点：recv () 返回值含义（TCP）

  • 返回 >0：成功读到数据
  • 返回 =0：对端正常关闭连接（发送 FIN）
  • 返回 =-1：出错或连接断开

三、UDP 详细知识点

1. UDP 是什么？

用户数据报协议

  • 无连接
  • 不保证可靠
  • 支持一对一、一对多、多对多
  • 基于数据报（一次发一个完整包）

2. UDP 特点

  • 不需要建立连接，发完就走
  • 不保证到达、不保证顺序
  • header 只有 8 字节，极快
  • 没有流量控制、拥塞控制

3. UDP 适用场景

  • 视频通话
  • 直播
  • 游戏
  • DNS
  • 实时传输（允许丢一点数据）

4. UDP 可靠吗？

本身不可靠，但可以在应用层自己实现可靠。

比如：QQ 文件传输就是 UDP + 自定义可靠机制。

四、高频面试题（你一定会遇到）

1. TCP 和 UDP 区别？

看最上面表格。

2. TCP 如何保证可靠？

序列号、ACK、超时重传、滑动窗口、拥塞控制。

3. 三次握手为什么是 3 次？

防止失效连接导致资源浪费。

4. 什么是 TIME_WAIT？

主动关闭方最后等待 2MSL，确保对方收到 ACK。

5. recv () 返回 0 代表什么？

对端已经正常关闭 TCP 连接。

6. 什么时候用 TCP，什么时候用 UDP？

  • 要可靠：TCP（网页、文件、邮件）
  • 要速度、实时性：UDP（直播、游戏、视频）
